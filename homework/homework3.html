Accessibility Checklist

Website Page Titles

After viewing your source code, it is good to see that your < title > tag is within the < head > tag. Titles are important as they are the first things that screen readers pick up and say when users switch between different web pages. They also are integral for orientation as users with many tabs open may need to navigate between different pages as well.

Furthermore, after accessing a few different web pages from your home page, I can safely say that your titles highlight the content on the immediate page as opposed to putting the organization's name first, which adequately summarizes the page's content as well as distinguishing it from other pages.

Results: 8 pages on the SCCLD website were evaluated for clear, unique HTML page titles. Each page has descriptive HTML title values. Each page also follows the format of "Current Page Content | Santa Clara County Library | Bibliocommons.""

Image Text Alternatives

If an image cannot load for whatever reason, image text alternatives are usually presented in lieu of the image itself. Screen readers also read the image text alternatives so it is vital to have alt text on images for the previous reasons.

Results: Only one image on the home page has alt text for it (the Santa Clara Library District logo). After looking at a few other pages as well, Bestsellers, Kids and Families, and the Online Library home page, I reached the same conclusion: that the only image with alt text on it was the logo itself. We recommend all photos to have alt text in order for patrons who use the website to know what each image is of and where it leads to.

Text:

Headings, Contrast Ratio ("color contrast") and Resizing Text are all tools that support navigation of the page in addition to remaining aesthetically pleasing.

Results - Headings After looking at the home page, the online library home page, kids and families, and bestsellers, we found that each page had only one h1 tag, which checks out as each page should ever only have one h1 tag so screen readers would know the hierarchy of the page.

Results - Contrast Ratio After looking at the same pages as before we noticed that the color ratios for text offered excellent contrast. The white background had black font, the black background had white font, and the dark teal had white font as well. The choices made here make visibility for text a lot easier on the eyes.

Results - Resizing Text SCCLD's website does a great job of text resizability all the way to 200% on desktop and the Google Chrome app on Android phones while maintaining an aesthetically pleasing appearance.

Interaction:

Keyboard access and visual focus

Using a mouse may require motor function and capability that not everyone has. Therefore, keyboard access should be readily available and keyboard focus should be denoted clearly through either a border or highlight.

Results: SCCLD's website is incredibly accessible via keyboard, allowing users to skip to either main navigation, search function, or content at the beginning of their options. Keyboard focus is also clearly denoted by an outline. Nice job!

Forms, labels, and errors (including Search fields):

Users are able to keyboard navigate to the different forms fields. Each form control has a label element with a for attribute that matches the value of the id attribute in the related control. For example:
  • Each form control has a label element with a for attribute that matches the value of the id attribute in the related control. For example:
    <label for="firstname">First name: </label>
    <input type="text" name="firstname" id="firstname" />
  • Each id is unique within the web page.

Results After testing the ask a librarian page, we were able to go to each form field and fill each one out using only the keyboard. Each form also had a unique id attached to it.

General:

Moving, Flashing, or Blinking Content: Moving, flashing, or blinking content includes carousels (example carousel), ads, videos, auto-updating stock tickers, scrolling news feeds, and more. While all of this content may help fit more information into one portion of the screen, users must be able to control any content that moves. It is not just a minor annoyance to wait for a certain piece of content to come around, some users may have visual processing disorders that would make it incredibly difficult to navigate the page.

Results: The lone image carousel on the home page near the top of the page has controls to pause the carousel as well as go back to an image from before (the three dots below the images that actually also make it much easier to select each one instead of having to keep going right or left).

Basic Structure Check: This basic check allows us to comprehend how some individual users will "view" the web page differently. In order to perform this check, we looked at the web page without images, styles, and layout. What we are left with is the barebones set up of the page and all of its links.

Results: We used the Web Developer extension for Google chrome to turn off all images and CSS styles. The lack of alt text for 99% of images hinders the webpage in this way. The lack of a "main" as well made navigation more difficult in this format. However, all the headings line up with the pages that follow them, for example, all of the book pages fall under the book heading, so folks using a screen reader might not have too much trouble navigating the home page.
